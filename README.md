# Fortigate Config Parser | Скрипт для парсинга конфигурационных файлов Fortigate
Преобразует имена объектов в группах адресов в конкретные IP-адреса и доменные имена
Собирает данные по группам в объектах типа config <name>

## Основные возможности
* Скрипт анализирует весь конфиг, чтобы найти определения для каждого участника группы.
* **Умное распознавание типов:**
    * Обрабатывает объекты вида `h-xxx.xxx.xxx.xxx` (хосты).
    * Извлекает значения из `set subnet` и `set fqdn`.
    * Автоматически определяет, является ли значение IP-адресом/подсетью или доменом.
* **Чистый вывод:** Формирует структурированный отчет, где для каждой группы отдельно перечислены IP-адреса и DNS-имена.
* **Безопасность:** Работает локально, не требует доступа к сети или самому устройству.

## Требования
* Python 3.6 или выше.
* Стандартная библиотека (используются модули `re`, `ipaddress`, `pathlib`).

## Инструкция по использованию
1.  Положите ваш файл конфигурации Fortigate в ту же папку, где находится скрипт.
2.  Запустите программу:
    ```bash
    python fortigate_parser.py
    ```
3.  Введите имя файла конфигурации, когда появится запрос.
4.  Введите целевой блок для парсинга. Например:
    `config firewall addrgrp`
5.  Результат будет сохранен в новый файл с названием вида `config_firewall_addrgrp_parsed.txt`.

## Пример работы

### Входные данные (в конфиге):
```text
edit "CORP_SERVERS"
    set member "h-10.0.0.5" "SRV-WEB-01"
next
end
...
edit "h-10.0.0.5"
    set subnet 10.0.0.10 255.255.255.255
next
edit "SRV-WEB-01"
    set fqdn "example.com"
next
```
### Полученые данные:
```text

    IP:
    10.0.0.10/32
    DNS:
    example.com
